Quality Checks

  1. HWE (p<10e-6)
  2. Reported sexes were confirmed
  3. Sample and variant call rate (>95%)
  4. Encoding of the pseudo-autosomal region within the X chromosome ( i.e. X merging)
 

LiftOver to build 38

 

  1. Position of the original files were changed to reflect the position within build 38 (i.e. lifted over) using UCSC LiftOver and their chain files. Unmapped variant, variant mapped to alternate contig, and mitochondrial variants were excluded.
  2.  Furthering the process, variants were aligned to the reference using an in-house script (Github) during this process A/T G/C palindromic variants were removed along with insertion and deletion. 
  3. Variants were also deduplicated using an in-house script (Github)  prioritizing well sequenced variants and renamed to Chr:Pos:Ref:Alt convention.

 

Merging

 

  1. variants with extreme differences in allele frequency established in a geographically aware test (script : Github) were removed.
  2. Samples were deduplicated (based on Kinship and sample ID) the most well sequenced was kept.

PCA

 

  1. Ancestry was established based on proximity of 1000 genomes samples within a PCA projection, samples with clear european ancestry were identified (K-NN, 10/10 within 6 dimension) any remaining outliers were removed.
  2. Ancestry was also established based on questionnaire answers.
  3. Individual with both clear DNA and questionnaire based European ancestry were selected in phenotype analysis.

 

Imputation

  1. Before imputation, all variant exhibiting strong allele frequency difference (>0.2) were identified using an in-house script (Github)
  2. All samples were separated randomly into two batches of 25000 (maximum number of samples in TOPMed at the time) with overlap. And imputed as batches.
  3. Batches were remerged and variants were filtered for Rsq > 0.3

Phenotype Association

  1. Regenie was distributed across a cluster per 50 phenotypes (github script)
  2. Covariants were the arrays (as categorical), the individual’s sex (as categorical), age, square(age)  and the 10 first PC.
  3. File preparation for Regenie were performed using the script provided in the github repository (github script). 
    1. Step 1 files were filtered using 
      • maf=0.01 (i.e. Minimum allele frequency)
      • geno=0.99 (i.e. Maximum Missing genotype frequency per-variant)
      • mind=0.99 (i.e. Maximum Missing genotype frequency per-sample)
      • HWE=1e-15 (i.e. threshold for Hardy-Weindberg Equilibrium)
    2. Step 2 files were unfiltered. 
      • Dosage (DS) was integrated into the analysis.
    3. Regenie Options
      • Binairy traits Options : --bt --firth --approx --pThresh 0.05
      • continuous traits option : --apply-rint

    PheWeb

    1. Variants with minimal allele count of 20 were loaded into PheWeb.
